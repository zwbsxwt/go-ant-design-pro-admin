# Feature Specification: 模块与菜单管理体验优化

**Feature Branch**: `011-module-menu-ux-optimization`

**Created**: 2026-07-24

**Status**: Completed

## User Scenarios & Testing

### User Story 1 - 平铺切换业务模块

已登录用户需要在右上角以平铺标签形式看到可用模块，并能一键切换当前模块，让左侧菜单只展示当前模块下的授权菜单。

**Acceptance Scenarios**

1. 首次登录时，右上角默认选中 `系统设置` 或第一个可见模块。
2. 多个模块可见时，右上角按排序平铺展示，当前模块高亮。
3. 点击另一个模块后，左侧菜单立即切换到该模块下的可见菜单。

### User Story 2 - 维护模块分组

管理员需要像维护分组一样维护模块，支持新增、编辑、隐藏、显示、启用、禁用、排序和删除。

**Acceptance Scenarios**

1. 隐藏模块后，该模块不再出现在右上角模块切换中，但直接访问已授权页面仍可进入。
2. 禁用模块后，该模块不再出现在右上角，也不再返回其下菜单。
3. 删除非空模块时，管理员必须选择目标模块，系统先迁移下属菜单再删除模块。
4. 不允许删除最后一个启用且未隐藏模块。

### User Story 3 - 优化菜单管理

管理员需要维护菜单是否在导航中展示，并能批量迁移菜单所属模块。

**Acceptance Scenarios**

1. 隐藏菜单后，该菜单不显示在左侧菜单中，但直接访问仍可进入。
2. 禁用菜单后，该菜单不显示在左侧菜单中，直接访问按无权限处理。
3. 菜单管理页支持按模块筛选，并可批量选择菜单迁移到目标模块。
4. 菜单管理页和模块管理页中文文案正常显示。

## Requirements

- **FR-001**: 右上角模块切换 MUST 使用平铺标签或 segmented control，不再使用下拉选择器。
- **FR-002**: `Module` MUST 支持 `hidden` 字段。
- **FR-003**: `Menu` MUST 支持 `hidden` 字段。
- **FR-004**: `currentUser.modules` MUST 只返回启用且未隐藏的模块。
- **FR-005**: `currentUser.menus` MUST 不返回禁用菜单。
- **FR-006**: 左侧导航 MUST 过滤隐藏菜单。
- **FR-007**: 模块删除 MUST 支持迁移下属菜单到目标模块后删除。
- **FR-008**: 菜单管理 MUST 支持批量迁移菜单到目标模块。
- **FR-009**: 隐藏 MUST 只影响导航展示，不撤销权限，不阻断直接访问。
- **FR-010**: 禁用 MUST 影响可用性，禁用模块或菜单不参与 currentUser 导航与访问。
- **FR-011**: 不改变角色权限仍基于菜单/按钮的模型。
- **FR-012**: 不改变 Higress 鉴权，网关仍只做 `/api/*` 转发。
- **FR-013**: 所有新建和修改文本文件 MUST 保持 UTF-8。

## Success Criteria

- **SC-001**: admin 登录后右上角模块以平铺形式显示。
- **SC-002**: 隐藏模块不会出现在右上角。
- **SC-003**: 隐藏菜单不会出现在左侧菜单，但直接访问仍可进入。
- **SC-004**: 禁用菜单直接访问显示 403 或无权限。
- **SC-005**: 删除非空模块时选择目标模块可完成迁移和删除。
- **SC-006**: 菜单批量迁移后，刷新 currentUser 可看到新的模块菜单归属。

## Assumptions

- 本次作为 010 的增量优化，新建 011，不覆盖 010 历史。
- 模块数量本期按少量设计，不做搜索、折叠或更多菜单。
- 隐藏和禁用是两个独立概念。
