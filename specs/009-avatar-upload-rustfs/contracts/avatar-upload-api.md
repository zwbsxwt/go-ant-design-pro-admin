# Contract: Avatar Upload API

## Scope

This contract extends the existing current-user profile API. The operation is authenticated and acts only on the signed-in user.

## Endpoint

### `POST /api/profile/avatar`

Upload and bind the current user's avatar.

### Request

Content type: `multipart/form-data`

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `file` | file | yes | PNG, JPEG, or WebP; max 2 MB |

### Success Response

HTTP status: `200`

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "avatar": "http://storage.example/go-ant-design-pro-admin/avatars/user-admin/avatar.png",
    "profile": {
      "id": "user-admin",
      "username": "admin",
      "display_name": "系统管理员",
      "avatar": "http://storage.example/go-ant-design-pro-admin/avatars/user-admin/avatar.png",
      "email": "admin@example.com",
      "phone": "",
      "status": "ACTIVE",
      "role_codes": ["admin"]
    }
  }
}
```

### Error Responses

| Case | HTTP Status | Error Code | Message Meaning |
| --- | --- | --- | --- |
| Not signed in | `401` | `UNAUTHORIZED` | 用户未登录或登录态失效 |
| Missing file | `400` | `INVALID_ARGUMENT` | 未选择头像文件 |
| Unsupported type | `400` | `INVALID_ARGUMENT` | 头像仅支持 PNG、JPEG、WebP |
| Oversized file | `400` | `INVALID_ARGUMENT` | 头像大小不能超过 2 MB |
| Storage unavailable | `503` | `STORAGE_UNAVAILABLE` | 对象存储不可用或配置错误 |
| Save failed | `500` | `PROFILE_SAVE_FAILED` | 头像已上传但用户资料保存失败 |

## Existing Response Extensions

### `GET /api/profile`

`data.avatar` MUST return the latest avatar URL after a successful upload.

### `GET /api/currentUser`

`data.avatar` MUST return the latest avatar URL after a successful upload.

## Frontend Contract

- Personal center uses a single avatar upload control.
- Frontend validates file type and file size before sending.
- Frontend still treats backend validation as authoritative.
- On success, frontend refreshes profile data and `initialState.currentUser`.
- On failure, frontend shows the backend error message and keeps the old avatar.

## Security Contract

- Access key and secret key are never sent to the browser.
- Uploaded object key is generated by backend.
- User-provided filename is not used as the final object key.
- Users can only update their own avatar through this endpoint.
- Higress remains a route-forwarding layer for `/api/*`.
