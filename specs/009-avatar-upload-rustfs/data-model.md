# Data Model: RustFS Avatar Upload And Display

## Entity: Avatar Object

Represents an uploaded avatar image stored in RustFS/S3-compatible object storage.

### Fields

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `bucket` | string | yes | Default example: `go-ant-design-pro-admin` |
| `object_key` | string | yes | Recommended format: `avatars/{user_id}/{uuid}.{ext}` |
| `content_type` | string | yes | One of `image/png`, `image/jpeg`, `image/webp` |
| `size_bytes` | integer | yes | Must be greater than 0 and no more than 2 MB |
| `public_url` | string | yes | Stored into user avatar field after successful upload |
| `created_at` | datetime | no | Not persisted in first version unless object metadata table is added later |

### Validation Rules

- File extension and detected content type must both be compatible with supported image formats.
- File size must be no more than 2 MB.
- Object key must be generated by the backend and must not trust user-provided filenames.
- Public URL must not contain access key, secret key, or private token.

## Entity: Current User Profile

Existing current-user profile with avatar value updated after upload.

### Fields Touched

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | string | yes | Current authenticated user ID |
| `avatar` | string | no | Browser-displayable avatar URL |
| `display_name` | string | no | Existing profile field, unchanged by upload |
| `username` | string | yes | Existing profile field, unchanged by upload |

### State Transition

```text
No avatar or old avatar
-> Valid upload accepted
-> Object stored in RustFS
-> system_users.avatar updated
-> currentUser/profile returns new avatar URL
```

## Entity: Object Storage Configuration

Runtime-only settings for RustFS/S3-compatible storage.

### Fields

| Field | Required | Notes |
| --- | --- | --- |
| `endpoint` | yes | Example shape: `http://host:9000`; do not commit private endpoints as required defaults |
| `region` | yes | RustFS local default can use `us-east-1` |
| `access_key` | yes | Secret; environment or private config only |
| `secret_key` | yes | Secret; environment or private config only |
| `bucket` | yes | Default template bucket name can be `go-ant-design-pro-admin` |
| `force_path_style` | yes | Required for the provided RustFS endpoint style |
| `public_base_url` | no | If set, compose avatar URLs from this base; otherwise use endpoint/bucket style URL |

### Relationships

- One current user profile references one latest avatar URL.
- One user may have multiple historical objects in storage, but only the latest URL is referenced by `system_users.avatar`.
- Object storage configuration is not persisted in MySQL.
